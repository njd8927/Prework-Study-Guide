HTML

  • The head element contains information about the webpage (metadata)
  • The body element represents the visible content shown to the user
  • To add an image // img src="File link goes here"
  • Header elements must be used in order from H1 to H6
  • Unordered lists are bullet points. Each bullet point must be wrapped with an li tag

CSS

  • A margin indicates how much space I want around the outside of an element
  • A padding indicates how much space I want around the content inside of an element
  • Use a comma to combine multiple selectors into a single CSS rule
  • Common CSS tags include: text-align, background-color, margin-left or right, font-size, box-shadow, border

Git

  • git status // checks what branch we are currently on
  • git checkout -b branch-name // creates a new branch and switches to it
  • git status // checks which branch currently working in
  • git add -A // commits all files in the working branch
  • git commit -m "message goes here" // associates a message to a commit
  • git must preceded every comment
  • git pull origin main // ensure current branch is up to date before opening a pull request
  • git push origin main // pushes up all changes made

Javascript

  • Semicolon must be used at the end of every line
  • Strings must be in quotation marks
  • console.log to print
  • Else if statements use === to determine if the code is truthy // if (topic === "Javascript"){ console.log ("Let's Study Javascript");}
  • Arrays are a single variable that hold a group of data // var shapes = ["triangle", "square", "pentagon", "circle"]
  • Index: location of the data item within an array. First item starts with 0 // Index number goes inside [] after the variable name // console.log (shapes[0]);
  • For Loops: for(var x = 0; x < shapes.length; x++) { console.log(shapes[x])}
  • x & 0 are default // x++ is a shortcut for x+1 // .length is a special property always used after the variable name.
  • Commenting Out is to make code inactive without deleting it // To comment out multiple lines of code enter /* at beginning and */ at end of code block.
  • Functions: use function keyword, the name of the function, (), {, code block, } // function helloWorld() { console.log("Hello World, I am a Function!) }
